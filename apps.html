gkLinks

サーバーも DB も無し、HTML 1ファイルで動く高機能リンク管理アプリ。
IT 開発者・PM の業務効率化を本気で考えた 24 機能を盛り込みました。

  • カテゴリ・パネル階層管理 / タグ / 全文検索（AND/OR/除外）
  • 🤖 AI プロンプトライブラリ（18種・Claude Code / Copilot 連携）
  • 🧩 コードスニペット集（18種・Java/TS/C#/VB.NET/SQL ほか）
  • 📊 ダッシュボード（統計・タグクラウド・ヒートマップ）
  • 📋 デイリースタンドアップ自動生成 / 📅 期限リマインダー
  • ⏱ PR レビュータイマー / 工数記録（CSV 出力）
  • 📝 ノートブック（議事録・ADR・設計書テンプレート）
  • ⌘ Cmd Palette (Ctrl+K) / 🎤 音声検索 / 📷 QR 生成
  • 🎨 8 テーマ切替（Dark Editor / Soft Gold / Paper Premium / Light / Forest / Ocean / Cyber / Sakura）
🏢 社内サーバー設置で全社員配布 gkLinks.html をイントラサーバー (IIS / nginx / Apache / 共有ドライブ) に置けば、全社員が同じ URL から開けます。
ライセンス料・月額・ユーザー数課金: 0円。SaaS でよくある「人数 × 月額」が発生しません。
データは各社員のブラウザ内 (LocalStorage) に個別保存、外部送信なし。共通リンク集は gkLinks_starter.json として一発配布できます。
動作環境 Windows 11 ／ Chrome・Edge（最新版推奨）
ZIP を解凍して HTML をダブルクリックで起動、もしくは社内サーバーに配置して URL アクセス

gkUI

日本人エンジニアのための WEB UI ライブラリ。
海外のUIライブラリで毎回ゼロから書いていた「日本特有の入力要件」を1ファイルで解決します。

  • 📅 和暦/西暦 DatePicker（令和・平成・昭和・大正・明治、7種フォーマット対応）
  • 📮 郵便番号 → 住所 自動入力（zipcloud API）
  • 📝 フリガナ自動入力（compositionupdate ベース・Microsoft IME / Google IME 動作確認）
  • ⌨️ IME 入力支援（全角ひらがな/カタカナ/半角数字/英数）
  • 0️⃣ ゼロ埋めコード入力（Excel貼付の前ゼロ消失問題に対応）
  • 💴 金額単位混在入力（「1,234千円」「12.3億」を内部で円に正規化）
  • 🎯 jQuery 並みに簡単: <script src="gkUI.js"> + data-gk-* 属性で結線
🎯 「これ、絶対売ってないよね」を目指して 海外ライブラリ (Material UI / Tailwind / Ant Design) に無く、既存日本系UIにも無い、
日本の業務システム開発の現場で毎回ゼロから書いてきた機能を網羅していきます。
v0.3 時点で 6 機能、v0.4 以降「税率切替」「インボイス番号」「全銀フォーマット」等を順次追加。
動作環境 Windows 11 / Chrome・Edge（最新版推奨）
HTML 1ファイル動作・<script> 組み込みの両対応・社内サーバー配布対応

gkReport

システム開発寄りの高速・高精度な日本帳票エンジンを、HTML 1ファイルで。
Excel / PDF / 直接印刷の3形態出力に対応します。

  • 📄 5 種テンプレート（請求書・見積書・納品書・仕様書・議事録）
  • 🖨️ A4 縦/横印刷 CSS（@page・改ページ・ヘッダ/フッタ固定）
  • 🏯 押印欄レイアウト（担当・主任・課長・部長・社印 可変）
  • 📅 和暦自動表示（gkUI 連携・令和8年4月29日 等）
  • 💴 消費税(10%)自動計算・小計・合計
  • 🎯 リアルタイム編集プレビュー（左ペイン入力 → 右ペイン即反映）
  • 📦 v0.2 予定: Excel テンプレート対応 / PDF出力 / Google Spreadsheet 対応
💎 商用帳票エンジンの軽量・低価格な選択肢 数十万〜数百万円規模の商用帳票エンジンが多い中、gkReport は システム開発寄りに設計。
サーバー不要・クライアントサイド完結で軽快、HTML+CSSで帳票を直接制御することでテンプレ自由度を確保。
中小企業・受託SI で「シンプルに業務帳票を回したい」現場向けの価格帯を目指します。
動作環境 Windows 11 / Chrome・Edge（最新版推奨）
HTML 1ファイル動作・<script> 組み込みの両対応・社内サーバー配布対応
