      1. はじめに

      📱 gkLinks とは

      gkLinks は、よく使うリンクを一元管理し、効率的にアクセスするためのオフライン完結型ツールです。1ファイル HTML として動作し、データはお使いのブラウザのローカルストレージに保存されます。

      💾 データの保存場所

      • 全データは localStorage（ブラウザ内）に保存
      • 主要キー: HBA_PANEL_CATS_v5.1（カテゴリ・パネル）, gkLinks_NOTEBOOKS（ノート）, gkLinks_AI_TPLS（AIテンプレ）, gkLinks_SNIPPETS（スニペット）, gkLinks_PROJECTS（プロジェクト一覧）など
      • ブラウザのデータを消去すると失われるので、定期的に「データ管理」タブから JSON エクスポートしてください

      🚀 スターターセット

      カテゴリが1つも無い状態では案内画面が出ます。「データ管理」タブの「🌐 スターターセットを追加」で、検索・SNS・生成AI の3カテゴリ（計14件）を一括投入できます。

      2. 基本操作

      📁 カテゴリ管理

      1. 「メイン」タブで「➕ カテゴリ追加」ボタン
      2. カテゴリ名・ヘッダー色・フォント色を入力
      3. カテゴリヘッダー右の ✎ で編集、🗑 で削除
      4. カテゴリ間ドラッグで順序入替え可能
      5. v1.7.3: ⚙️ プロジェクト管理ボタンも追加されています

      📄 パネル管理

      1. カテゴリ内の「＋ このカテゴリにパネル追加」、または「メイン」タブの「📄 パネル追加」
      2. タイトル・URL・画像URL・タグ・説明文・期限を入力
      3. パネル右上の ✎ で編集、🗑 で削除、☆ でお気に入り
      4. パネルをドラッグして同一/別カテゴリへ移動可能

      🏷️ タグ・期限・お気に入り

      • タグはカンマ区切りで複数指定可（業務, 朝イチ, 緊急）
      • ヘッダーの「タグで絞込...」から複数選択 AND/OR フィルタ
      • 期限を設定したパネルはダッシュボードの「期限近いタスク」に集約
      • ☆クリックで「お気に入り」カテゴリへ。常に最上段に固定表示

      ♿ アクセシビリティ・キーボード操作

      • Tab でフォーカス移動、Enter で実行、Esc でモーダルを閉じる
      • Ctrl+K で Cmd Palette（後述）
      3. 業務改善機能

      🤖 AIプロンプトライブラリ（18種、v1.4.0+）

      「AIツール」タブから利用。Claude Code / GitHub Copilot 連携を想定した、コードレビュー・テスト生成・リファクタリング・要件整理などのテンプレートが18件プリセット。コピー＆カスタマイズして使えます。新規追加・編集・削除可能。

      🧩 コードスニペット集（18種、v1.6.0+）

      「スニペット」タブから利用。よく使うコードパターン（fetch, async/await, regex, debounce, etc）を言語別にカテゴライズ。コピーボタンで即使用。タイトル・本文・タグで横断検索可能。

      📋 デイリースタンドアップ（v1.5.0+）

      「ダッシュボード」タブの上部から、昨日 / 今週のサマリを Markdown 形式で自動生成。よく開いたリンク・編集パネル・AI 使用状況を集計し、チャットツール や 他のドキュメントツール に貼り付け可能。

      ⏱ PR レビュータイマー / 工数記録（v1.7.0+）

      • PR タイマー: ダッシュボードで PR を登録すると経過時間カウント。レビュー漏れ防止
      • 工数記録: タスクごとにタイマー開始/停止 → CSV エクスポート可能

      📅 期限リマインダー（v1.7.0+）

      パネルに期限を設定すると、ダッシュボードの「期限近いタスク」に今日〜7日以内のものを自動抽出。

      🎯 プロジェクトコンテキスト（v1.5.0、v1.7.3 で管理UI追加）

      • ヘッダーのセレクタで「プロジェクト」を切替えると、タグに @プロジェクト名 を含むパネルだけが表示される絞込モード
      • パネル編集モーダルの「📌 プロジェクトに紐付ける」で現在のプロジェクトタグをワンクリック追加
      • v1.7.3: ⚙️ ボタンから既存プロジェクトのリネーム・削除が可能。削除時はパネルからのタグ自動除去も選択可

      🎫 チケット番号自動検出（v1.6.0+）

      パネルの説明文や URL に JIRA-1234 / BLG-5678 / #9999 形式があると自動検出して、リンク化やコピー支援が可能（チケット連携設定で URL テンプレを登録）。

      4. 高度な機能

      ⌘ Cmd Palette（Ctrl+K、v1.1.0+）

      VS Code 風のコマンドパレット。パネル横断検索 + よく使う操作（カテゴリ追加・タブ切替・エクスポート等）を一気に実行できます。Ctrl+K で起動、↑↓で選択、Enter で確定、Esc で閉じる。

      🎤 音声検索（v1.1.0+）

      ヘッダー検索欄の右の 🎤 ボタンをクリック → 音声を認識して検索クエリに反映。Chrome / Edge のみ対応（Web Speech API）。

      📷 QR コード生成（v1.1.0+）

      パネル右クリック → 「📱 QRコードを表示」で URL の QR コードを生成。スマホで読み取って即アクセス。

      🌐 Favicon 自動取得（v1.1.0+）

      パネルの画像URLが空でも URL から Google Favicon サービス経由で自動取得・表示。

      📝 ノートブック（v1.7.0+）

      「ノートブック」タブから複数ノートを管理。Quill リッチテキストエディタ + 6 種テンプレート（議事録・ADR・設計書・1on1・KPT・週報）。各ノートはローカル保存、PDF/HTML エクスポート可能。

      🔗 URL 共有（v1.2.0+）

      ヘッダーの URL 共有ボタンから現在のカテゴリ全体を Base64 化して URL に埋め込む短縮 URL を生成。チームでセット共有や設定移動に便利。

      📅 利用期間別表示

      「表示設定」タブの利用期間ラジオで、今週 / 今月 / 先月以前 別にパネルを抽出・一括削除可能。整理に便利。

      5. データ管理

      📤 JSON エクスポート / インポート

      • 「データ管理」タブの「📤 設定をエクスポート」でタイムスタンプ付き JSON を保存
      • 「📥 設定をインポート」で JSON を読み込み（既存データ上書き）

      🌐 ブラウザブックマーク（HTML）互換

      Chrome/Edge/Firefox 形式のブックマーク HTML をインポート可能。階層付きフォルダ構造をカテゴリにマッピング。

      💾 バックアップ・復元

      JSON は十分にバックアップ。別マシン・別ブラウザへの移行も同 JSON のインポートで可能。

      ⚠️ 全クリア機能: 「データ管理」タブの「全クリア」は全カテゴリ・パネル・お気に入り・履歴を十分に削除します。元に戻せません。実行前に基本的にエクスポートを。

      7. トラブルシューティング

      ❓ データが消えた

      1. ブラウザの開発者ツール（F12）→ Application → Local Storage を開く
      2. HBA_PANEL_CATS_v5.1 の値を確認。[] なら誤クリアの可能性
      3. 過去にエクスポートした JSON があれば「データ管理」→「設定インポート」から復元
      4. シークレット / プライベートウィンドウで開いた場合、ウィンドウを閉じると消える仕様です

      ❓ 音声検索が動かない

      Web Speech API の制限により Chrome / Edge / Safari のみ対応。Firefox は未サポート。マイク使用許可が必要です。

      ❓ QR コードが出ない

      QR コード生成にはネットワーク接続が必要です（外部 QR 生成 API を使用）。オフラインでは表示できません。

      ❓ 一括で開いたら一部しか開かない

      ブラウザのポップアップブロック機能による制限です。アドレスバー右端のポップアップアイコンから「このサイトのポップアップを許可」を選択してください。

      ❓ カテゴリが他タブにも表示される

      v1.7.3 で「メイン」「表示設定」タブ以外では非表示になりました。古い v1.7.2 以前を開いている場合はキャッシュをクリアしてください。

      ❓ プロジェクトがリネーム / 削除できない

      v1.7.3 でヘッダーのプロジェクトセレクタの隣に ⚙️ プロジェクト管理 ボタンを追加しました。そこからリネーム・削除が可能です。
